CUI 中文字体与排版系统

字体

中文字体解决方案

CUI 使用以下字体声明来为英文字体与中文字体全局性的提供更好的兼容度以及美观程度：

font-family: 'Helvetica Neue', Helvetica, Tahoma, Arial, STXihei, "华文细黑", "Microsoft YaHei", "微软雅黑", SimSun, "宋体", Heiti, "黑体", sans-serif;

字体辅助Class

西文字体辅助Class

根据西文字体常见分类：衬线（serif）、非衬线（sans-serif），CUI 提供下面两个字体声明作为辅助Class：

.font-serif {
  font-family: Georgia, "Times New Roman", serif;
}
.font-sans {
  font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
}

等宽字体辅助Class

CUI 提供了一个等宽字体的辅助Class，这种字体多见于代码字体或计算机的输出文字所用：

.font-mono {
  font-family: Menlo, Consolas, Courier, sans-serif;
}

中文字体辅助Class

针对Web 上常用的四种中文字体： 黑体、宋体、楷体、仿宋，CUI 提供了相应的兼容全平台（OSX、Windows、Linux）的辅助Class：

.font-hei{
  font-family: "Helvetica Neue", Helvetica, Arial, "Hiragino Sans GB", "PingFang SC", "Lantinghei SC Extralight", FZLTXHK--GBK1-0, "Lantinghei SC", "Heiti SC", "Microsoft Yahei", "SimHei", "Source Han Sans SC", "Source Han Sans CN", "Wenquanyi Micro Hei", "WenQuanYi Zen Hei", "WenQuanYi Zen Hei Sharp", "Droid Sans Fallback", sans-serif;
}
.font-song {
  font-family: Georgia, "Times New Roman", "Songti SC Regular", STSongti-SC-Regular, "Songti SC", STSong, NSimSun, SimSun, "AR PL New Sung", "TW\-Sung", "WenQuanYi Bitmap Song", "AR PL UMing CN", "AR PL UMing HK", "AR PL UMing TW", "AR PL UMing TW MBE", serif;
}
.font-kai {
  font-family: Georgia, "Times New Roman", "Kaiti SC Regular", STKaiTi-SC-Regular, "Kaiti SC", STKaiti, Kai, Kaiti, KaiTi_GB2312, "AR PL UKai CN", "AR PL UKai HK", "AR PL UKai TW", "AR PL UKai TW MBE", "AR PL KaitiM GB", DFKai-SB, "TW\-Kai", serif;
}
.font-fang-song {
  font-family: Georgia, "Times New Roman", STFangsong, FangSong, FangSong_GB2312, "CWTEX\-F", serif;
}

注意：以上中文字体所使用的字体声明中，根据字体所属的衬线类别，分别使用了与上面西文字体辅助Class 一样的字体声明顺序来兼容中文字体中可能出现的衬线、非衬线西文字体。

页面主体：body & p

CUI 将全局 font-size 设置为 14px，line-height 设置为 1.428。 这些属性直接赋予 <body> 元素以及继承body 样式的元素。 另外，<p> （段落）元素还被设置了等于 1/2 行高（即 10px）的底部外边距（margin）。

计算机中最基本的描述状态的概念是位，取值只能是0和1，然后通过不同的位的组合来达到存储数据，表达数据的作用，其中，“位”又有一个称呼叫做“比特”。而在存储和处理数据时，普遍使用8个比特作为最小的存储和处理单元，而这8个比特组成的单元称为字节。

ASCII 字符集是由美国国家标准协会ANSI 指定的标准，旨在规定常用字符的集合以及每个字符对应的编码，并以此希望统一字符-字节的对应关系，避免同一段字节在不同计算机显示出来的字符不一致的问题。

ASCII 字符集使用一个字节来包含95个可打印字符（0x20-0x7E）和33个控制字符（0x00-0x19、0x7F），总共128 个字符，而这128个字符（2的7次方，对应的编码十六进制区段是：0x00-0x7F）占用了一个字节的后7位。也正因为这个原因，ASCII 规定最前面的1位统一定为0，只允许使用最后的7位来对应字符。
